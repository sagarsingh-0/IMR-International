I already have an existing full-stack web application. I want to integrate a **Consent-Based Anonymous User Behaviour Monitoring System with an Advanced Admin Analytics Dashboard** into it WITHOUT breaking any existing functionality.

---

## 🎯 OBJECTIVE

Build a privacy-first analytics system where:

* Tracking starts ONLY after user consent
* No personal data (PII) is stored
* All analytics are visible ONLY in an admin dashboard
* Admin can view DAILY, WEEKLY, and MONTHLY reports
* System identifies user interest trends

---

## ⚠️ IMPORTANT RULES

* Do NOT modify existing business logic
* Add all analytics features as a separate module/service
* Do NOT store:

  * name, email, IP address, device fingerprint
* Use anonymization (hashing + UUID)
* Follow privacy-by-design principles

---

# 🧩 FRONTEND INTEGRATION

## 1. Consent Banner

* Show on first visit
* विकल्प: Accept / Reject
* Store consent in localStorage
* Add "Withdraw Consent" option

---

## 2. Tracking System (Silent)

Create a reusable tracker utility:

Example:

```js
if (localStorage.getItem("consent") === "accepted") {
  trackEvent("page_view", window.location.pathname);
}
```

Track:

* Page views
* Button clicks
* Navigation
* Session duration

---

## 3. Session Handling

* Generate UUID sessionId
* Store temporarily (no persistence)
* Send with each event

---

# 🧱 BACKEND INTEGRATION

## 1. APIs

POST /api/consent
→ Save consent status

POST /api/events
→ Receive event
→ Hash sessionId (SHA-256)
→ Store anonymized data

---

## 2. Analytics APIs (TIME-BASED)

GET /api/analytics/daily
→ Today’s visits, clicks, top pages

GET /api/analytics/weekly
→ Last 7 days visits + daily breakdown

GET /api/analytics/monthly
→ Last 30 days visits + weekly breakdown

GET /api/analytics/summary
→ Overall stats

---

## 3. Database Schema

### events

* id
* hashed_session_id
* event_type
* page
* timestamp

### consent_logs

* id
* status
* timestamp

---

## 4. Aggregation Logic

* Group by day → daily reports
* Group by week → weekly reports
* Group by month → monthly reports

Calculate:

* total visits

## 5. Interest Detection

Return:
